# Git Workflow for Media Tracker

## Quick Commands

### Check Status
```bash
git status
```

### Stage All Changes
```bash
git add .
```

### Commit Changes
```bash
git commit -m "Your commit message here"
```

### Push to GitHub
```bash
git push origin main
```

### Pull Latest Changes
```bash
git pull origin main
```

## Typical Workflow

1. **Make your changes** in Cursor
2. **Check what changed:**
   ```bash
   git status
   git diff
   ```
3. **Stage changes:**
   ```bash
   git add .
   # Or stage specific files:
   git add path/to/file.py
   ```
4. **Commit:**
   ```bash
   git commit -m "Description of changes"
   ```
5. **Push:**
   ```bash
   git push origin main
   ```

## Branch Workflow (Optional)

For feature development:
```bash
# Create a new branch
git checkout -b feature-name

# Make changes and commit
git add .
git commit -m "Add feature"

# Push branch
git push origin feature-name

# Merge back to main (on GitHub or locally)
git checkout main
git merge feature-name
```

## Notes

- The repository is configured with authentication, so pushes should work directly from Cursor
- Always check `git status` before committing to see what will be included
- Write clear, descriptive commit messages
- The `.gitignore` file excludes database files, virtual environments, and other local files
